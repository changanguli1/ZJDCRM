## 三、必须完成的 V1.0 功能

### 1. 登录和账号安全

- 支持内部账号、手机号或邮箱登录。
- PBKDF2-SHA-256，随机盐，合理迭代次数。
- 数据库不保存明文密码。
- 会话令牌使用密码学安全随机值，数据库只保存哈希。
- Cookie 必须 `HttpOnly; Secure; SameSite=Lax; Path=/`。
- 连续 5 次登录失败，账号锁定 30 分钟。
- 登录成功、失败、锁定、退出、改密都记录日志。
- 禁用账号不能登录，已有会话也立即失效。
- 所有 mutation 必须做同源和 CSRF 验证。
- API 错误使用统一结构，不返回堆栈。

路由至少包括：

```text
POST /api/auth/login
GET  /api/auth/session
POST /api/auth/logout
POST /api/auth/change-password
```

### 2. RBAC 和数据权限

默认角色：

- 超级管理员：全部。
- 管理层：全部招商数据、看板、报表、导出审批。
- 招商主管：本团队数据、分配和转移。
- 招商人员：本人负责数据。
- 运营 / 综合岗：授权范围的数据、空间、导入和报表。

实现：

- 菜单权限。
- 按钮/动作权限。
- 数据范围：本人、本团队、全部、指定部门。
- 负责人和协作人的记录级读写权限。
- 列表、详情、统计、附件、导出必须复用同一个权限服务。
- 不能只靠前端隐藏。
- 对 IDOR 越权请求返回 404 或 403，不能泄漏对象信息。

### 3. 招商线索

线索必须覆盖设计规格和 PRD 中的全部字段：

- 线索名称。
- 企业名称和主营业务。
- 所属行业。
- 意向空间、面积。
- 获取意向日期。
- 预计落位日期。
- 招商阶段。
- 核心卡点。
- 渠道来源。
- 内部转介。
- 融资需求。
- 原办公/生产所在地。
- 流失原因。
- 财源完成情况。
- 预计产值、预计税收。
- 负责人。
- 标签。
- 附件。
- 实际落位空间、面积、日期。

功能：

- 新增、编辑、查看、分页、排序、筛选。
- 录入人默认负责人。
- Excel 导入未指定负责人时进入未分配池。
- 主管分配和负责人转移。
- 软删除与管理员恢复。
- 已签约、已落地禁止删除。
- 乐观锁 version，冲突返回 409。

### 4. 查重

- 企业名称规范化：空格、全半角、大小写、常见公司后缀。
- 活跃企业标准化名称唯一。
- 联系人手机号重复提示并阻止直接新增。
- 导入重复项进入失败明细。
- 冲突返回已有记录 ID，方便前端跳转。

### 5. 联系人

- 一条线索可维护多个联系人。
- 姓名、手机、电话、邮箱、职务、是否决策人、备注。
- 联系人继承线索数据权限。
- 进入“初步接触”前必须至少有一个联系人。

### 6. 跟进和时间线

跟进字段：

- 跟进方式。
- 跟进时间。
- 跟进内容。
- 客户反馈。
- 核心卡点。
- 下一步动作。
- 下次跟进时间。
- 附件。
- 可选同步变更阶段、预计落位时间和空间需求。

时间线统一展示：

- 跟进。
- 阶段变化。
- 分配和负责人转移。
- 空间匹配。
- 关键编辑。

### 7. 阶段状态机

默认阶段：

```text
新线索
已建档
初步接触
需求确认
重点跟进
考察洽谈
意向确认
签约推进
已签约
已注册 / 已落地
暂缓 / 流失
```

规则：

- 允许前进和后退。
- 每次变化必须填写原因。
- 阶段历史和时间线必须与线索更新原子写入。
- 初步接触必须有联系人。
- 暂缓/流失必须填写原因。
- 已注册/已落地必须填写实际空间、实际面积、落地日期、财源完成情况。
- 只有负责人、主管或授权人员可变更。

### 8. 空间资源

层级：

```text
园区 → 楼宇 → 楼层 → 空间
```

空间字段至少包括面积、可用面积、状态、预计释放日期、备注。

规则：

- 一条线索可关联多个备选空间。
- 一个空间可被多个线索意向。
- 意向不锁定空间。
- 运营/管理员维护空间状态。
- 被引用空间不可直接删除。
- 落地时记录实际空间与实际面积。

### 9. 附件

- 存在私有 R2。
- 随机对象键，不能信任原文件名。
- MIME 和扩展名双重验证。
- 限制文件大小。
- 下载前检查业务对象权限。
- 流式返回，不读取无限数据到内存。
- 业务删除后延迟清理 R2。

### 10. 站内通知

- 下次跟进。
- 线索分配。
- 重点项目阶段变化。
- 导出申请。
- 导出审批结果。
- 项目标记流失。
- 已读、全部已读。
- Cron 定期产生到期提醒，必须幂等。

### 11. 首页和报表

首页：

- 新增线索数。
- 阶段分布。
- 招商漏斗和转化率。
- 渠道来源。
- 人员负责线索与跟进次数。
- 重点项目。
- 预计落位面积。
- 已签约/已落地。
- 预计产值/税收。
- 财源完成情况。
- 空间状态。
- 待办提醒。

报表：

- 招商漏斗。
- 渠道分析。
- 人员绩效。
- 跟进效率。
- 空间面积和状态。
- 落地成果。

要求：

- 时间范围筛选。
- 所有统计服从数据权限。
- 图表可点击跳转到等价筛选的线索列表。
- 转化率分母为零时安全处理。

### 12. Excel 固定模板导入

模板：

- 招商线索。
- 企业和联系人。
- 空间资源。

实现：

- 固定 `.xlsx` 模板，有隐藏 metadata sheet 保存模板类型和版本。
- 校验表头、必填项、日期、数字、字典值、重复项。
- 1000 行以内分批处理。
- 任务可轮询进度。
- 展示成功数、失败数、每行失败原因。
- 处理 chunk 必须幂等，重试不能重复写入。
- 写导入审计。

### 13. 导出申请和审批

- 普通招商人员不可导出。
- 申请人选择对象、字段和筛选范围，原因必填。
- 固化申请范围与数据权限快照。
- 管理层审批；驳回原因必填。
- 批准后重新校验当前权限并生成文件。
- 文件存 R2。
- 下载令牌随机生成，数据库只保存哈希。
- 24 小时失效。
- 只能申请人或授权管理员下载。
- 申请、审批、生成、下载全部审计。

### 14. `/admin` 管理后台

入口：

```text
https://cfzzs.custard.top/admin
```

普通用户菜单不显示，但安全不能依赖隐藏路径。只有 `system:admin:access` 可进入。

后台必须管理：

- 员工和账号。
- 部门树。
- 角色和权限。
- 数据范围。
- 阶段、行业、渠道、标签、跟进方式、流失原因、空间状态等字典。
- 园区、楼宇、楼层、空间。
- 导入任务。
- 导出审批。
- 登录和操作审计。
- 已删除数据恢复。
- 网站名称、Logo、登录页文案、系统公告、看板开关。
- 备份记录。

保护规则：

- 不能删除最后一个有效超级管理员。
- 已引用字典项不能删除，只能禁用。
- 所有后台 mutation 写审计。

### 15. 初始管理员

- 用户名 `admin`。

- 初始化脚本读取 Secret、哈希后写数据库。

## 五、API 与工程要求

- 使用当前 Hono + Cloudflare Pages Functions。
- 使用 D1 参数绑定。
- R2 通过 binding，不从 Worker 内调用 Cloudflare REST API。
- 不使用全局可变请求状态。
- 所有 Promise 必须 await、return、void 或 `ctx.waitUntil()`。
- 安全关键日志必须 await。
- 使用生成的 Cloudflare binding 类型，不手写漂移的 Env。
- 新增 schema 必须用新 migration。
- API 使用统一 JSON envelope 和 requestId。
- 对大文件流式处理。
